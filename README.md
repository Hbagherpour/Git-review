# Git-review
This repository is a hands-on practice project for learning and reviewing core Git and GitHub concepts using VS Code on Windows.

What I Practiced
Throughout this project, I explored the following topics:

✅ Terminal & Environment

I started by understanding the difference between PowerShell and Git Bash terminals in VS Code, learning how each one works and how to set Git Bash as the default terminal. I also learned how to navigate in and out of the Python interactive interpreter (REPL) from the terminal.

✅ Git Configuration

I learned how to check my Git username, email, and default branch name using git config commands, and how to read the full configuration list without getting stuck in the pager.

✅ Repository Setup

I practiced initializing a new Git repository with git init, understanding what a .git folder means, and why Git commands only work inside a repository folder.

✅ Branching

I explored the concept of branches in Git, understanding the difference between master and main as default branch names, how to rename a branch, and how to create and switch between branches.

✅ Commits & Logs

I practiced staging files with git add, committing changes with git commit, and reviewing commit history using git log and git log --oneline.

✅ Pushing to GitHub

I connected my local repository to a remote GitHub repository and pushed my code using git remote add origin, git branch -M main, and git push -u origin main.

✅ Issues, Pull Requests & Merging

I learned how to create an Issue on GitHub to track and describe a change or task. I then worked on the change in a separate branch, opened a Pull Request to propose merging it into the main branch, reviewed the changes, and merged the Pull Request into main. This gave me a solid understanding of the standard GitHub collaboration workflow used in real-world projects.

✅ Syncing with GitHub

After merging the Pull Request on GitHub, I learned how to bring those latest changes back into my local repository on my laptop using git pull, keeping my local and remote repositories in sync.
